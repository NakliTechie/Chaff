How Chaff works

Chaff encrypts your message, then uses a language model to weave the ciphertext into natural-looking cover text. Your recipient — with the same model and the same password — runs it in reverse to get the message back. Everything happens in your browser.

Using it

  1. Load the model once (it’s cached afterward and works offline).
  2. You and your recipient set the same shared password. Agree on it through a separate, trusted channel — not in the same chat you’ll send the cover text through.
  3. Hide: type your secret, press Generate, and copy the cover text. Send it via any chat app, email, wherever.
  4. Reveal: paste the cover text you received and press Reveal.

Tips

  • Both sides must use the same password and the same bucket width (under Advanced).
  • If a reveal fails, regenerate the cover text, or nudge bucket width up a notch on both sides.
  • Paste as plain text — apps that auto-format or “smarten” quotes can alter the cover.
  • The built-in self-test (Advanced) confirms the core works without downloading the model.

What it does and doesn’t protect

Chaff hides that a message exists from casual reading and keyword filters. It is not undetectable: someone who has this exact model and specifically suspects steganography can detect it statistically. Your safety rests on the password staying secret and on not being individually targeted by a matched-model detector. Treat it as cover, not a guarantee.
