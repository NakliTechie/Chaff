Chaff

Hide a secret inside innocent-looking text, using a language model running entirely in your browser. Same model + same shared password on both ends. No server, no account, offline after the model loads.

Read this. Chaff makes cover that looks ordinary to a human or a keyword filter. It is not undetectable: an adversary who has this exact model and is looking for stego can detect it statistically. Your safety rests on them lacking the shared password and the suspicion to run a matched-model detector. Don't rely on it against a targeted, well-resourced adversary.

Advanced & self-test
The offline self-test round-trips random secrets through the exact same encryption + coding logic this page uses, with a deterministic mock model — proving the core works without downloading GPT-2.

Both people must use the same password AND the same bucketWidth. If a reveal fails, the cover text was altered in transit, the password/bucketWidth differ, or the two devices' model outputs diverge (widen bucketWidth and re-generate).

Runs Xenova/gpt2 in-browser via transformers.js (ONNX Runtime Web, WASM, fp32, single-thread — pinned for determinism). Encryption: PBKDF2 (600k) → AES-256-GCM, fails closed. The coding layer uses its own integer fixed-point softmax and a snap-to-grid bucketed rank coder so the two ends agree bit-for-bit. Open source, AGPL-3.0. This is an M0 spike: robust against casual review, not against a matched-model detector.
